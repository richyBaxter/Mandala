How I built the Bitcoin Spiral (and why)

A build log by houtini · open the live dashboard →

I built this for a slightly nerdy reason: I wanted to understand GitHub's built-in AI model library. Reading about a tool and shipping something with it are different sports, so I shipped something.

GitHub quietly ships a thing called GitHub Models: a catalogue of frontier models (OpenAI, Meta, DeepSeek, Mistral and friends) that you can call straight from a GitHub Action with nothing but your repo's own token. No API key to provision, no billing to set up, no separate account. It runs on a free, rate-limited tier that is more than generous for anything that isn't a production workload. I'd read the docs. I wanted to feel it work.

This project is here because I happen to like two things: cryptocurrency, and dynamic dashboards. Bitcoin gave me a data set with real signal and a genuinely engaged audience. Dashboards gave me the format. And the more I built, the more I landed on an opinion I'll happily defend to anyone who'll listen:

A live, public dashboard is one of the most underrated pieces of content a company can make, and almost nobody is making them.

Think about it. A blog post is read once and dies in the feed. A dashboard is a destination. It updates itself, so it stays relevant without anyone touching it. It earns links and embeds because it is genuinely useful. It proves you know your market far better than another listicle ever could, because you cannot fake a working model of the thing your customers care about. And the kicker: with the stack I'm about to describe, it costs nothing to run. Free hosting, free data, free inference. Whatever your business is (sim racing gear, coffee, conveyancing) there is a live number your audience cares about, and a dashboard is the content type that owns it.

So this repo is a prototype. It is the proof that the whole thing fits together and stays free. Here is how it actually works.

Zero backend, on purpose

There is no server. There is no database. There is not even a build step. The entire site is plain HTML, SVG and JavaScript that hydrates in your browser from free public APIs. Open the dashboard and your own browser fetches the price, the sentiment, the flows and the network stats, and renders them.

This matters more than it sounds. No backend means nothing to break at 3am, nothing to pay for, nothing to patch. It deploys to GitHub Pages straight from the repo. Every panel is written to fall back to "n/a" if a feed is rate-limited or blocked, so a flaky API never takes the page down. It just quietly shows a dash where a number would be.

The spiral

The hero image is generated by a single Python script (standard library only, zero dependencies). It plots Bitcoin's price against the Power Law on a polar map where one full turn is a four-year cycle. Angle is when in the cycle you are. Radius is price on a log scale. The orange band is the Power-Law channel that Bitcoin has tracked for fourteen years: floor, fair value, and the cycle-top band. It is, on purpose, a picture you can read in five seconds and then stare at for five minutes.

The dashboard

The dashboard is the live read. It pulls a dozen signals in the browser and renders stat tiles, a stack of charts (price versus the Power-Law channel, Pi Cycle, Mayer Multiple, Fear & Greed, taker-volume pressure, hashrate) and a verdict banner that blends six indicators into one plain-English call: Accumulate, Lean buy, Neutral, Lean sell, Distribute. The point is not to be clever. It is to take a beginner from "number go up?" to "here is where we actually sit in the cycle, and here is what that has historically meant."

The bit I'm proud of: an AI note that doesn't lie

Every six hours a GitHub Action wakes up, and this is where GitHub Models earns its place. The pipeline does five things in order:

  1. Refresh the data. Pull real monthly closes from Binance, scrape spot-ETF flows from Farside, grab institutional headlines server-side.
  2. Redraw the spiral from the fresh data.
  3. Compute the signals. Four orthogonal reads, all in plain Python: valuation (where price sits in the channel, plus how that ranks against every month since 2011 as a base rate), trajectory (30 and 90-day change, plus the Mayer Multiple), sentiment (Fear & Greed, then versus a month ago) and demand (7 and 30-day ETF net flow).
  4. Write the note. GitHub Models turns those numbers into three or four calm sentences.
  5. Commit and deploy. The refreshed data and image are committed back to the repo and Pages redeploys. The site is never more than six hours stale, and no human touched it.

The discipline in those last two steps is the whole trick, and it is the thing most people get wrong when they bolt an LLM onto data. The model is a writer, not a calculator. Every figure it sees is computed in Python first, and the prompt forbids it from inventing or predicting numbers. So the prose always agrees with the dashboard, it never hallucinates a price target, and if the live feeds are down it falls back to a computed note rather than making something up. An LLM is brilliant at turning a pile of numbers into a sentence a human wants to read. It is terrible at being the source of those numbers. Keep those jobs separate and the result is reliable. Blur them and you ship a confident liar.

What it costs

Nothing. GitHub Pages hosting: free. The public price, sentiment and network APIs: free. GitHub Models inference: free, on a rate-limited tier that four calls a day does not come close to troubling. The only thing I spent was an afternoon.

Take it

This is open source under the MIT licence. Fork it, rip out Bitcoin, point the prompt builder at your own numbers, and you have a self-updating, AI-narrated dashboard for whatever metric your audience actually cares about. That was the experiment. As far as I'm concerned, it worked.
